## Workflow (VERY IMPORTANT)

### Do NOT work directly on `main`

### Always follow these steps:

1. **Create a new branch**

   * Click вҖңCurrent BranchвҖқ вҶ’ вҖңNew BranchвҖқ
   * Name it something like:

     * `feature/homepage`
     * `feature/about-page`
     * `fix/navbar`

2. **Make your changes**

   * Add/edit files in the correct folders

3. **Commit your changes**

   * Write a clear message:

     * вҖңAdd homepage layoutвҖқ
     * вҖңFix navigation linksвҖқ

4. **Push your branch**

   * Click вҖңPush originвҖқ

5. **Create a Pull Request (PR)**

   * Open the repo on GitHub
   * Click вҖңCompare & pull requestвҖқ
   * Add a short description

6. **Wait for review before merging**

## File Organization Rules

* Keep files in the correct folders:

  * CSS вҶ’ `/css`
  * JS вҶ’ `/js`
  * Images вҶ’ `/images`
  * Pages вҶ’ `/pages`

* Only these files go in the root:

  * `index.html`
  * `README.md`
